03The second brain

It doesn't just store notes. It files them.

A folder of notes gets messier as it grows. This gets smarter — because every new thing is filed in four places at once and linked to everything it relates to. Four kinds of page, one job each.

sources

Where it came from

One page per document you feed it — a summary of that single thing, linked back to the untouched original.

"Sales call · Meridian", "P&L · September"
entities

Things with names

Clients, competitors, the people you deal with, the tools you run on. If it could wear a name badge, it's an entity.

"Meridian Group", "Sarah Chen"
concepts

Ideas and topics

The things you can't point at — pricing, churn, runway. The problems you keep circling back to.

"pricing power", "churn", "cash runway"
syntheses

What you concluded

Answers built by comparing across many sources — the thinking that would otherwise die in a chat window.

"Why we lost 3 deals last quarter"
one brain · everything linked hover to focus · drag to move
sources entities concepts syntheses
One sales call, filed five ways:
The transcript stays untouched — that's the receipt.
A source page captures what was actually said on that call.
The account and the buyer each get an entity page — every future mention lands there too.
The themes that came up — pricing power, churn — get concept pages.
Three months later, comparing that call against your win/loss reviews and churn survey produces a synthesis: why you actually lost last quarter.
That last one is the point. No single call told you the answer — it only appears when the notes are connected. File the call under the account alone and the pattern stays invisible.
04Why it's a system, not a chatbot

Six things chatbot memory can't do.

Modern AI assistants remember, schedule, and run skills — the feature gap is closed. The difference isn't features. It's the substrate: what the knowledge is made of, and who owns it.

Auditable

Every change to the business brain is a git commit — diffable, revertible, attributable.

Chatbot memory: a black box that just "builds up."

Owned & portable

Plain markdown you keep. It survives any plan change — and can be packaged and shared as an asset.

Chatbot memory: captive to one vendor's cloud.

Structured, not emergent

An enforced schema — sources cited, contradictions flagged. It knows, it doesn't just hope it remembered.

Chatbot memory: emergent — hope it caught it.

One brain, not silos

A single cross-linked knowledge base. Every note connects — nothing is trapped in a standalone thread.

Chatbot memory: scoped per project, per session.

Multi-person & tiered

A shared, role-tiered repository your whole team draws from — with the sensitive parts scoped by role.

Chatbot memory: single-user, per account.

Integration control

Your own scoped API keys, wired through code you can read — you decide what each connection can touch.

Chatbot memory: a black-box connector holds your keys.
05How it's built

A method, not a one-off setup.

An AI-OS is grown, not installed. The build follows three moves — think, decide, ship — and repeats: one process automated, one piece of knowledge captured, every week.

01 · MINDSET

How to think

Spot the repetitive, rules-based work worth handing to AI — and ask "to what extent could AI do this?" before defaulting to the manual way.

02 · METHOD

How to decide

Scope one candidate tightly. Define the inputs, the steps, the output, and the guardrails before a line is built.

03 · MACHINE

How to build

Codify it as a skill in the repo, run it on real work, and let it compound. Ship one, then the next.
